# Module Development

模块是 Agent Harness Template 承载具体能力的边界。模板核心保持通用，具体逻辑应放在 `modules/{module_name}/` 内。

## 创建模块

使用脚手架：

```bash
python3 cli/scaffold_module.py sample_agent
```

脚手架会阻止覆盖已有模块。

## 标准结构

```text
modules/sample_agent/
  README.md
  module.yaml
  agent.yaml
  prompts/
    system.md
  services/
    sample_agent_service.py
  skills/
    .gitkeep
  evals/
    .gitkeep
```

## 文件职责

- `module.yaml`：模块元信息。
- `agent.yaml`：Agent 元信息。
- `prompts/`：模块提示词。
- `services/`：模块服务逻辑。
- `skills/`：模块局部技能说明或资产。
- `evals/`：模块局部评测资产。
- `README.md`：模块说明。

## Schema 参考

V0.1.8 为模块配置补齐了可扫描的最小 contract：

- `module.yaml` 应参考 `schemas/module.schema.json`。
- `agent.yaml` 应参考 `schemas/agent.schema.json`。

模块配置可以在模块目录内扩展，但不应把模块自己的逻辑写进底座核心目录。

## module.yaml

```yaml
id: sample_agent
name: Sample Agent
version: 0.1.0
description: Generic module scaffold generated by Agent Harness Template.
entrypoint: modules.sample_agent.services.sample_agent_service:execute
enabled: true
capabilities:
  - scaffold
default_agent: sample_agent_agent
```

核心字段：

- `id`：模块唯一标识。
- `entrypoint`：执行入口，格式为 `module.path:function`。
- `enabled`：是否允许 registry 发现和选择。
- `capabilities`：模块声明的通用能力。
- `default_agent`：默认 Agent id。

## agent.yaml

```yaml
id: sample_agent_agent
module_id: sample_agent
name: Sample Agent Agent
description: Generic agent scaffold for this module.
system_prompt_path: prompts/system.md
skills: []
tools: []
workflow: service
provider: mock
```

核心字段：

- `system_prompt_path`：模块内 system prompt 路径。
- `skills`：Agent 使用的 skill id。
- `tools`：Agent 使用的 tool id。
- `workflow`：执行方式。
- `provider`：默认 provider id。

## Execution Contract

模块入口函数应实现：

```python
def execute(input_text, context):
    ...
```

入口返回 `AgentExecutionResult`，至少包含：

- `output`：最终输出。
- `steps`：可观察的执行步骤。
- `metadata`：可选元数据。

`context` 至少包含：

- `module_id`
- `agent_id`
- `run_id`
- `metadata`

## 开发建议

- 先明确模块边界。
- 保持模板核心业务无关。
- 把模块自己的提示词、服务、技能和评测资产放在模块目录内。
- 需要接入主链路时，先确认模块 manifest 和 `execute` 入口符合 contract。
