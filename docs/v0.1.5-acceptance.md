# V0.1.5 Acceptance 测试与验收增强

V0.1.5 只增强测试覆盖、验收脚本和验收文档，不增加业务功能。

## 覆盖范围

- Run persistence：验证 Run、Step、RunEvent 写入数据库，并保持 API 返回兼容。
- File / Artifact persistence：验证 File 与 Artifact 写入数据库，并保持 API 返回兼容。
- Knowledge persistence：验证 Document、Chunk 写入数据库，retrieve 返回 citation。
- Schema validation：验证 `schemas/*.schema.json` 是合法 JSON，且包含核心契约字段。
- Scaffold：验证 module scaffold 能创建合法模块、拒绝非法名称、拒绝覆盖已有模块。
- Business-neutral guard：验证底座核心目录没有业务词污染。

## 数据隔离

后端测试使用 `apps/api/tests/conftest.py` 设置测试专用 `DATABASE_URL` 和 `LOCAL_STORAGE_DIR`。

每个测试前都会重建测试数据库表，避免测试之间互相污染。Scaffold 测试使用临时目录，不会写入真实 `modules/`。

## 完整验收命令

```bash
make test-api
python3 scripts/check_business_terms.py
for f in schemas/*.schema.json; do python3 -m json.tool "$f" >/dev/null || exit 1; done
cd apps/web && npm run build
```

## 预期结果

- 后端测试全部通过。
- 业务词污染检查无失败。
- 所有 schema JSON 格式合法。
- 前端构建通过。

## 不包含

- 不创建业务 Agent。
- 不修改现有 API 路径。
- 不接真实外部模型。
- 不实现 embedding、向量数据库、多模态、人审、权限或多租户。
