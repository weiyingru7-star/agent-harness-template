# V0.1.6 Provider Acceptance

V0.1.6 完善 AI Runtime Provider 框架，不创建业务 Agent，不改变 `/api/runs` 主链路。

## 已完成能力

- 统一 LLM Provider 最小接口。
- 保留 mock provider 作为默认 provider。
- 完善 OpenAI-compatible provider 的配置项和基础错误处理。
- 新增 Provider Router，根据请求或环境变量选择 provider。
- `/api/llm/smoke` 保持兼容。
- 测试不依赖真实 API Key。

## Provider 配置

```env
AI_PROVIDER=mock
AI_BASE_URL=
AI_API_KEY=
AI_MODEL=gpt-4o-mini
AI_TIMEOUT=30
```

默认使用 `mock`。真实 provider 未配置时应返回明确错误，不影响测试和主链路。

## 验收命令

```bash
make test-api
npm run build --prefix apps/web
python3 scripts/check_business_terms.py
```

Mock smoke：

```bash
curl -X POST http://localhost:8005/api/llm/smoke \
  -H 'Content-Type: application/json' \
  -d '{"prompt":"hello","provider":"mock"}'
```

Structured mock smoke：

```bash
curl -X POST http://localhost:8005/api/llm/smoke \
  -H 'Content-Type: application/json' \
  -d '{"prompt":"hello","provider":"mock","structured":true}'
```

OpenAI-compatible 未配置检查：

```bash
curl -X POST http://localhost:8005/api/llm/smoke \
  -H 'Content-Type: application/json' \
  -d '{"prompt":"hello","provider":"openai_compatible"}'
```

预期返回 400，并说明缺少 provider 配置。

## 当前限制

- 不实现 embedding。
- 不实现向量数据库。
- 不实现多模态。
- 不实现生产级模型路由。
- 不让测试依赖真实外部模型。
