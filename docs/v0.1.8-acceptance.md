# V0.1.8 Module Registry Acceptance

V0.1.8 增强 Module Registry 和 Agent Execution Contract，不创建新的业务模块，不改变现有 API 路径。

## 已完成能力

- 自动扫描 `modules/*/module.yaml`。
- 自动读取 `modules/*/agent.yaml`。
- 非法模块 manifest 会被跳过并记录 warning。
- `demo_agent` 提供标准 `execute(input_text, context)` 入口。
- `/api/modules` 返回模块规范字段。
- `/api/runs` 不传 `module_id` 时默认运行 `demo_agent`。
- `/api/runs` 传 `module_id=demo_agent` 时可运行。
- 脚手架模板生成的 `module.yaml` / `agent.yaml` 符合新 contract。

## 验收命令

```bash
make test-api
npm run build --prefix apps/web
python3 scripts/check_business_terms.py
git diff --check
```

API 验收：

```bash
curl http://localhost:8005/api/modules

curl -X POST http://localhost:8005/api/runs \
  -H 'Content-Type: application/json' \
  -d '{"input":"hello"}'

curl -X POST http://localhost:8005/api/runs \
  -H 'Content-Type: application/json' \
  -d '{"input":"hello","module_id":"demo_agent"}'
```

## 当前限制

- 不创建业务模块。
- 不实现远程模块市场。
- 不实现权限系统。
- 不实现多租户。
- 不改变 `/api/runs` 路径。
